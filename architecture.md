# architecture.md — 입력 → 처리 → 출력 〔D5 핵심〕

> `자동심사 현황분석 Agent`의 입력, 처리, 출력, 사람 승인 지점, 실패 시 동작을 한 장으로 정리한 구조 문서.

| 섹션 | 내용 |
|---|---|
| Goal | 운영관리자가 자동지급 현황의 월말 변화, segment 변화, 자동지급률, 처리효율, 안정 상태를 빠르게 읽고 상위 보고할 수 있게 한다. |
| Input | 지급데이터 · 자동지급 대상군 월말 스냅샷 · segment 변경 이력 · 자동지급률 데이터 · 비용환산 기준/단가 · 월별 자동지급 현황/리포트 이력 · 조회기간 필터 값 |
| Process | ① 월말 스냅샷 수집 → ② 전월 대비 비율 변화 계산(자동지급 대상군 기준) → ③ segment 변화 탐지 및 변화 설명 생성 → ④ 처리효율/성과 계산(ROI 보고용) → ⑤ 안정/변화 상태 판정 → ⑥ 운영모니터링 화면 갱신 → ⑦ 월간 리포트 자동 생성 → ⑧ 매월 1일 자동 발송 → ⑨ 리포트보관함 누적 저장 |
| Tools | 데이터 저장소(지급데이터/이력) · 리포트 생성 모듈 · 발송 모듈 · 운영모니터링 화면 · 리포트보관함 · 조회 필터 · 상태 판정 규칙 |
| Output | 운영모니터링 화면 · 자동지급률 그래프 · 성과 표시 · segment 변화 설명 · 안정/변화 상태 라벨 · 월간 리포트 파일 · 자동발송 결과 · 리포트 재다운로드 |
| HITL | 운영관리자가 월간 리포트와 대시보드 해석을 최종 확인한다. 안정 상태 정의와 임계치 조정은 운영 거버넌스 판단에 따른다. segment 변화 원인 분류 기준은 운영정책에 따라 보완한다. |
| Failure | 월말 데이터 미확정 → `미확정` 상태로 표시하고 최종 리포트 생성 보류 / 분모 정의 불일치 → 비율 계산 중단 후 확인 필요 / segment 이력 누락 → 변화 설명은 생성하되 `원인 확인 필요`로 표기 / 발송 실패 → 재발송 큐에 남기고 보관함에는 생성본만 저장 |
| Cost | 월 1회 리포트 생성 + 운영자 조회 중심. 정량 비용은 아직 확인 필요. 보고서 생성은 배치형, 조회는 경량 계산으로 설계한다. |

## To-be Workflow

1. **데이터 수집**
   - 지급데이터와 자동지급 대상군 월말 스냅샷을 적재한다.
   - segment 변경 이력과 비용환산 기준을 함께 연결한다.

2. **월말 비교**
   - 월말 기준으로 전월 대비 변화를 계산한다.
   - `비율 변화`를 우선 계산하고, 기준 모수는 `자동지급 대상군`으로 둔다.

3. **변화 설명 생성**
   - segment가 어떻게 바뀌었는지 설명 문장을 만든다.
   - 변화 전/후 차이를 함께 보여준다.
   - 원인 정보가 부족하면 `확인 필요`로 남긴다.

4. **성과/안정 판정**
   - ROI 보고용 성과를 계산한다.
   - 운영효율 중심으로 표시한다.
   - 변화가 없거나 허용 범위 내면 `안정` 상태로 명시한다.

5. **운영모니터링 반영**
   - 대시보드 첫 화면에서 `그래프 / 자동지급률 / 성과 / 안정 상태`를 고정 노출한다.
   - 사용자는 기본적으로 운영모니터링 화면에서 시작한다.
   - 필요 시 기간 필터로 반기 외 구간도 조회한다.

6. **월간 리포트 생성**
   - 대시보드에서 사용한 근거를 바탕으로 월간 리포트를 자동 생성한다.
   - 리포트는 월별 현황, segment 변화, 증감률, 성과, 안정 상태를 포함한다.

7. **자동 발송 및 보관**
   - 매월 1일 구독 계정 목록으로 자동 발송한다.
   - 생성된 리포트는 리포트보관함에 누적 저장한다.
   - 사용자는 보관함에서 재조회 및 재다운로드할 수 있다.

## Data Flow Summary

`지급데이터 / 월말 스냅샷 / segment 이력 / 비용환산 기준`
→ `월말 비교 및 변화 계산`
→ `변화 설명 / 성과 / 안정 상태 판정`
→ `운영모니터링 화면`
→ `월간 리포트 자동 생성`
→ `자동 발송`
→ `리포트보관함 저장`

## 운영 원칙

- 성과는 판단값이 아니라 ROI 보고용이다.
- 변화가 없어도 지표는 계속 노출하고, 상태는 `안정`으로 표시한다.
- 월말 스냅샷 기준이 기본이며, 조회기간 필터는 보조 기능이다.
- 청구유형 분류, 최종 지급 확정, 지급금액 단독 확정은 하지 않는다.
- 사람 심사가 필요한 건을 자동지급으로 강하게 밀어주는 흐름은 만들지 않는다.

# PRD — 자동심사 현황분석 Agent
**Version**: v0.1  
**Date**: 2026-06-15  
**Status**: DRAFT

## 1) 문제
현재 보험금 자동지급 운영은 시스템 rule로 지급 처리가 되더라도, **현황 분석과 보고는 사람이 수작업으로 처리**한다.  
운영관리자는 월말 기준으로 자동지급 대상군의 변화, segment 변화, 자동지급률, 처리효율을 확인하고 상위 보고해야 하지만, 전월 대비 변화와 원인 설명을 매번 직접 정리해야 한다.  
이로 인해 운영 판단과 보고가 늦어지고, 변화가 없을 때도 안정 상태를 일관되게 보여주는 구조가 부족하다.

### 문제 근거
- **사실**: 월간 현황 점검과 보고서 작성은 수작업으로 이뤄진다.
- **사실**: 운영관리자는 월말 스냅샷 기준의 전월 대비 변화와 segment 변화를 설명해야 한다.
- **추정**: 이 과정에서 분석과 보고 준비 시간이 반복적으로 소요된다.
- **확인 필요**: 월간 리포트 작성 시간, 수작업 쿼리/점검 횟수, 변화 설명에 소요되는 평균 시간을 수치로 확보해야 한다.

## 2) 사용자
### 1차 사용자
- **운영관리자**
  - 자동지급 운영 현황을 월말 스냅샷 기준으로 확인한다.
  - 전월 대비 변화와 segment 변화를 설명한다.
  - ROI 보고용 성과와 안정 상태를 상위 보고에 사용한다.

### 향후 확대 사용자
- 심사 Auditor
- 심사담당자

### 사용 상황

- 필요 시 리포트보관함에서 월간 리포트를 조회한다.
- 월말 기준으로 전월 대비 어떤 변화가 있었는지 확인한다.

## 3) 범위
### 만들 것
- 운영모니터링 화면
- 리포트보관함
- 자동지급률 그래프
- 성과 표시
- segment 변화 설명
- 월말 기준 전월 대비 변화 설명
- 안정 상태 표시
- 조회기간 필터
- 월간 리포트 자동 생성
- 월간 리포트 자동 발송
- 월간 리포트 재다운로드

### 1차 핵심 기능
- 자동지급 대상군의 월말 스냅샷 추이 확인
- 자동지급률과 처리효율 확인
- 전월 대비 비율 변화 확인
- segment 변화 설명
- 안정/변화 상태를 함께 노출
- 월간 리포트 자동 생성 및 보관

### D5 인계 메모
- **입력**: 지급데이터, 월말 스냅샷, segment 변경 이력, 비용환산 기준
- **처리**: 월말 비교 → 비율 변화 계산 → segment 변화 설명 → 안정/변화 상태 판정 → 월간 리포트 생성
- **출력**: 운영모니터링 화면, 리포트보관함, 월간 리포트, 자동발송 결과

### 비고
- 성과는 판단값이 아니라 **ROI 보고용**이다.
- 성과 해석은 **운영효율 중심**이다.
- 변화가 없어도 지표는 계속 노출하고, 상태는 `안정`으로 명시한다.
- 실제 개발에서는 지급데이터와 월말 스냅샷을 분리된 테이블과 조회 조건으로 관리한다.

### 입력 데이터
- 지급데이터
- 자동지급 대상군 월말 스냅샷
- segment 변경 이력
- 자동지급률 데이터
- 성과 산식에 필요한 비용환산 기준 및 단가
- 월별 자동지급 현황 및 리포트 이력

### 출력 형식
- 운영모니터링 화면
- 리포트보관함 화면
- 월간 리포트
- 리포트 자동발송 결과
- 안정/변화 상태 표시

### 사람 승인 지점
- 운영관리자가 월간 리포트와 대시보드 해석을 최종 확인한다.
- 안정 상태 정의와 임계치 조정은 운영 거버넌스 판단에 따른다.
- `segment 변화`의 원인 분류 기준은 운영정책에 따라 보완할 수 있다.

### 데이터 계약 메모
- 지급데이터: 보험금 지급 관련 원천 데이터
- 월말 스냅샷: 월말 기준의 비교 시점 데이터
- 자동지급 대상군: 비율 계산의 분모
- segment: 운영 구분 값
- segment 변경 이력: 변화 설명의 근거
- 자동지급률: 자동지급 처리된 비율
- 처리효율/성과: ROI 보고용 지표
- 안정 상태: 변화가 없거나 허용 범위 내인 상태

## 4) 완료조건
다음이 가능하면 1차 버전이 완료된 것으로 본다.
- 운영관리자가 대시보드에서 자동지급 대상군의 월말 변화와 segment 변화를 확인할 수 있다.
- 자동지급률, 성과, 안정 상태를 한 화면에서 함께 볼 수 있다.

- 리포트를 보관함에서 재조회하고 재다운로드할 수 있다.
- 상위 보고에 쓸 수 있을 정도로 변화와 상태를 설명할 수 있다.

## 6) 리스크
### 사용자/운영 리스크
- 운영관리자가 실제로 대시보드와 리포트를 계속 사용하지 않을 수 있다.
- 지표가 너무 많으면 첫 화면이 복잡해질 수 있다.
- 변화가 없어도 계속 노출되는 지표가 정보 과잉으로 느껴질 수 있다.

### 데이터/정의 리스크
- `자동지급 대상군`의 분모 정의가 불명확하면 비율 해석이 흔들릴 수 있다.
- `segment 변화`를 설명할 때 원인 분류 기준이 부족할 수 있다.
- `비용환산 기준`과 `처리효율`의 해석이 조직 내에서 다를 수 있다.
- 월말 스냅샷만으로 설명할 때 월중 급변을 놓칠 수 있다.  
  - 현재는 월말 기준만으로 처리하는 방향으로 정리했지만, 추후 운영상 보완이 필요할 수 있다.

### 운영 거버넌스 리스크
- 안정 상태 정의와 임계치 조정 권한이 불명확하면 운영 기준이 흔들릴 수 있다.
- 상위 보고자와 운영관리자가 같은 수치를 다르게 해석할 수 있다.

### 후보 검증 질문
1. `비율 변화`의 분모는 `자동지급 대상군` 기준으로 고정해도 되는가?
2. `처리효율`은 비용환산 기준으로 표현할 때 상위보고자도 같은 의미로 읽는가?
3. `segment 변화`는 변경 사실만이 아니라 변경 전후 차이까지 보여줘야 하는가?
4. 월말 스냅샷만으로 운영 상황을 설명해도 되는가?
5. 안정 상태는 색/상태로 명시했을 때 정보 과잉 없이 이해되는가?

- 상위 보고에 필요한 ROI 관련 지표를 재가공하는 시간이 줄어든다.

- 대시보드 조회 수
- 리포트 열람 수
- 리포트 재다운로드 수
- 월별 자동지급률 추이
- 월별 성과 추이
- segment 변화 발생 건수
- 안정/변화 상태 비율

### 확인 필요 지표
- 운영관리자 사용 빈도
- 보고서 작성 시간 절감량
- 리포트 자동발송 성공률
- 월말 스냅샷 데이터 확정 시점
- 임계치/상태 정의 변경 이력

### 검증 기준
- 운영관리자가 운영모니터링 화면에서 먼저 시작한다.
- 리포트보관함은 필요 시 여는 보조 진입점으로 동작한다.
- 월말 기준 전월 대비 변화와 segment 변화가 설명 가능하다.
- `안정` 상태가 반복 노출돼도 운영 판단을 방해하지 않는다.

## 8) 기능명세
### 8-1. 운영모니터링 화면
**목적**
- 운영관리자가 월말 기준 현황을 가장 먼저 읽는 기본 화면이다.

**첫 화면 구성**
- 상단 헤더: 화면명, 기준월, 조회기간, 데이터 확정 여부
- KPI 카드: 자동지급률, 처리효율, 자동지급 건수, 제외/인심사 건수, 상태 라벨
- 추이 그래프: 선택한 기간의 월별 자동지급률과 처리효율 추이
- 변화 요약: 전월 대비 변화와 segment 변화 요약
- 보조 영역: 확인 필요 항목과 리포트 진입 버튼

**첫 화면 노출 순서**
1. 기준월과 데이터 확정 여부를 먼저 보여준다.
2. 핵심 KPI 카드 5개를 한 줄 또는 한 묶음으로 보여준다.
3. 가운데 영역에 월별 추이 그래프를 배치한다.
4. 하단에 전월 대비 변화와 segment 변화 설명을 붙인다.
5. 마지막에 리포트보관함 또는 월간 리포트로 이동하는 진입점을 둔다.

**기본 조회 규칙**
- 기본 조회기간은 반기다.
- 필터로 기간을 바꿀 수 있다.
- 그래프 기간은 6개월 단위로 늘리거나 줄일 수 있다.
- 월말 스냅샷이 미확정이면 `미확정` 상태를 먼저 보여준다.

**표시 원칙**
- 숫자는 먼저 보여주고, 해석 문구는 그 아래에 붙인다.
- 변화가 없더라도 지표는 숨기지 않는다.
- 변화가 있으면 `전월 대비 증가/감소`와 함께 표시한다.
- 상태 라벨은 색보다 먼저 텍스트로 명시한다.
- 확인 필요 항목은 그래프보다 아래에 배치한다.

**상태 라벨 종류**
- `안정`: 전월 대비 변화가 없거나 허용 범위 내인 경우
- `변화`: 전월 대비 유의미한 변동이 있는 경우
- `미확정`: 해당 월의 월말 스냅샷이 아직 확정되지 않은 경우

**데이터 확정 여부 의미**
- `Y`: 해당 월의 월말 스냅샷이 확보되고 확정되어 공식 비교 기준으로 사용할 수 있는 상태
- `N`: 해당 월의 월말 스냅샷이 아직 확정되지 않아 공식 월간 리포트 기준으로는 사용할 수 없는 상태

**기준월 노출 규칙**
- 기본 화면은 가장 최근에 확정된 월말 스냅샷을 기준월로 보여준다.
- 예를 들어 오늘이 6월 16일이고 6월 월말 스냅샷이 아직 확정되지 않았다면, 기본 화면은 5월까지의 확정 데이터만 보여준다.
- 6월 데이터가 임시로 적재되어 있더라도 `미확정`으로만 보여주고, 공식 리포트와 기본 비교 기준에는 사용하지 않는다.

### 8-2. 리포트보관함
**목적**
- 월간 리포트를 재조회하고 재다운로드하는 보관 영역이다.

**목록 구성**
- 기준월
- 생성일
- 발송일
- 리포트 상태
- 파일명
- 마지막 조회일
- 재다운로드 버튼
- 상세 보기 버튼

**리포트 상태 종류**
- `생성됨`: 리포트는 생성되었지만 아직 발송 전인 상태
- `발송됨`: 리포트가 구독자에게 발송 완료된 상태
- `미발송`: 리포트 생성은 되었지만 발송 실패 또는 발송 보류 상태
- `재생성 필요`: 기준 데이터가 바뀌어 현재 버전을 다시 만들어야 하는 상태

**동작 규칙**
- 발송된 리포트는 보관함에서 다시 열 수 있어야 한다.
- 월말 스냅샷이 확정되지 않은 경우에는 리포트를 생성하지 않는다.
- `미발송` 리포트는 재발송할 수 있어야 한다.
- `재생성 필요` 상태는 기존 파일을 유지하되 최신 기준으로 다시 만들 수 있게 표시한다.
- 상세 보기에서는 기준월, 비교월, 상태, 주요 요약문, 발송 이력을 함께 보여준다.

**기본 정렬**
- 최신 기준월이 위에 오도록 내림차순 정렬한다.
- 같은 기준월 안에서는 생성일이 최신인 버전을 위로 둔다.

**보조 규칙**
- 재다운로드는 권한이 있는 사용자만 가능하게 둔다.
- 보관함의 기본 진입점은 운영모니터링 화면에서 연결한다.
- 보관 기간은 운영정책에 따라 별도 설정 가능하게 둔다.

### 8-3. 월간 리포트
**목적**
- 운영관리자가 상위 보고에 바로 쓸 수 있도록 월말 기준 현황을 짧고 일관되게 요약하는 문서다.

**타이틀 구조**
- 최상위 타이틀: `월간 자동심사 현황 리포트`
- 대제목: `자동지급 현황 개요`, `자동지급 유형 분석`, `자동지급 제외유형 분석`
- 소제목: 각 대제목 아래에 `무엇을 보여주는지`를 한 줄로 적는다
- 보조 문구: 기준월, 비교월, 데이터 확정 여부를 타이틀 바로 아래에 둔다

**필수 포함 항목**
- 기준월과 비교월
- 자동지급률
- 전월 대비 변화
- segment 변화 요약
- 처리효율/성과
- 안정 상태 여부
- 운영관리자용 짧은 해석 문구

**본문 구조**
1. 요약 영역
   - 기준월, 비교월, 데이터 확정 여부
   - 자동지급률, 처리효율, 상태 라벨
2. 변화 영역
   - 전월 대비 자동지급률 변화
   - 전월 대비 처리효율 변화
   - 증가/감소한 핵심 항목
3. segment 영역
   - segment 변경 전/후
   - 변경 건수 또는 비중
   - 변화 사유 요약
4. 운영 해석 영역
   - 이번 달을 `안정` 또는 `변화`로 본 이유
   - 확인 필요 항목
5. 다음 조치 영역
   - 운영자가 추가로 확인해야 할 점
   - 재발송 또는 재조회가 필요한 경우 안내

**섹션별 내용**
1. 자동지급 현황 개요
   - 해당 월의 자동지급률
   - 접수채널별 자동지급 비율
   - 진료구분별 자동지급 비율
2. 자동지급 유형 분석
   - 상위 5개 유형
   - 전월 대비 증감률
3. 자동지급 제외유형 분석
   - 제외유형별 비율
   - 전월 대비 증감률

**본문 작성 원칙**
- 첫 문단에서 결론을 먼저 쓴다.
- 숫자와 해석을 분리해서 쓴다.
- segment 변화는 변경 전/후를 같이 적는다.
- 확인 필요 항목은 마지막에 분리해서 적는다.
- 공식 리포트는 확정월 기준으로만 생성한다.
- 시각화가 의미 전달에 더 유리하면 표나 차트를 함께 넣는다.
- 차트는 설명 문구를 대신하지 않고 보조한다.

**시각화 추가 기준**
- 값의 구성비를 빠르게 보여줘야 하면 파이/도넛 차트를 사용한다.
- 추세나 증감을 보여줘야 하면 막대/라인 차트를 사용한다.
- 상위 5개 유형처럼 순위가 중요하면 막대 차트를 우선한다.
- 접수채널별, 진료구분별 비율처럼 비교가 중요하면 그룹 막대나 누적 막대 차트를 우선한다.
- 해석이 차트보다 복잡하면 차트를 생략하고 표로 적는다.

**문구 원칙**
- 결과 설명은 짧고 직접적으로 쓴다.
- 변화 원인이 불명확하면 `확인 필요`를 붙인다.
- 보고서 문구는 화면 문구와 같은 근거를 사용한다.

### 8-4. 상태 표시 규칙
**상태 정의**
- `안정`: 전월 대비 변화가 없거나 허용 범위 내인 경우
- `변화`: 전월 대비 유의미한 변동이 있는 경우
- `미확정`: 월말 데이터가 아직 확정되지 않았거나 비교가 불완전한 경우

**표시 규칙**
- 상태는 숫자 옆에 함께 보여준다.
- 안정 상태도 숨기지 않고 명시적으로 표기한다.
- 미확정 상태는 리포트 발송과 분리해서 처리한다.

### 8-5. 조회 및 필터 규칙
**필수 필터**
- 조회기간
- 기준월

**보조 필터**
- 상태
- segment
- 리포트 상태

**규칙**
- 기본값은 반기다.
- 기간 필터는 사용자가 쉽게 바꿀 수 있어야 한다.
- 필터 결과는 화면과 리포트 모두 같은 기준을 사용한다.

# Monthly Report Template v3 — 자동심사 현황분석 Agent

> `report_template_upgrade.docx`와 `report_sample_upgrade.docx`를 기준으로 정리한 월간 리포트 작성 기준이다.  
> 이 문서에서 **모든 글씨는 검정색**으로 작성한다.  
> Agent는 데이터로 문구를 채우되, `report_template_upgrade.docx`의 폰트·크기·줄간격·정렬을 그대로 따른다.

## 1) 작성 원칙

- 문단 구성은 항상 `소제목 -> ☐ 항목 -> - 설명 -> (표)` 흐름을 따른다.
- 기본 폰트는 `바탕체`다.
- 제목은 24pt, 본문과 표 본문은 12pt, 유형표 헤더와 본문은 10pt로 본다.
- 발행년월과 팀명은 우측 정렬, 문서 제목은 가운데 정렬한다.
- 제목에는 underline을 넣는다.
- 표 첫행은 모두 가운데 정렬, 첫행 아래부터는 `유형명`만 왼쪽 정렬하고 나머지는 가운데 정렬한다.
- 모든 텍스트 색상은 `#000000`으로 유지한다.
- 표 다음에는 다음 문단을 시작하기 전 빈 줄을 한 줄 둔다.
- 숫자는 단순 재진술이 아니라, 비교와 해석이 보이도록 쓴다.
- 표는 행 수를 줄이더라도 핵심 변화가 보이게 만든다.
- 전월 대비 변화, 청구 특성 변화, 유형 변화는 반드시 함께 읽히게 한다.
- 추측은 추측이라고 표시하고, 원인 단정은 피한다.
- 화면과 리포트는 같은 지표명과 같은 상태명을 사용한다.
- 요약 표의 `전월比`는 가능한 한 숫자로 채운다.
- 유형 표는 `순위 / 코드 / 유형명 / 비중 / 당월 / 전월 / 증감` 7열 구조로 본다.

## 2) 색상 규칙

### 고정영역
- 문서 제목
- 섹션 번호와 소제목
- `☐` 체크 모양
- `- 기 간:` 같은 고정 라벨
- 표의 열 제목
- 운영팀명
- Action Item의 기본 골격 문구

### 가변영역
- 발행년월
- 기간 값
- 각 섹션의 설명 문구
- 해석 결과 문장
- 표의 숫자 값과 변화 설명
- 상황에 따라 추가/삭제되는 인사이트 문장

## 3) 문서 구조

### 3-1. 표지

고정:
- `자동지급 현황 월간 보고`
- `보험금심사팀`

가변:
- 발행년월 표시

작성 기준:
- 발행년월은 `"'" + YY.MM + "月"` 형태로 표시한다.
- 표지에서 이미 문서의 기준시점을 읽을 수 있어야 한다.
- 발행년월과 기간 표시는 템플릿의 정렬과 크기를 그대로 따른다.

### 3-2. 1. 요약

#### ☐ 전체 처리현황

고정:
- `- 기 간:`
- `항목 / 당월 / 전월比` 표 구조

가변:
- 보고 기간
- 표의 수치
- 전체 현황 해석 문장

반드시 들어갈 표 항목:
- 총 청구건수
- 자동지급 대상건수
- 자동지급 건수
- 제외/인심사 건수
- 자동지급률
- 처리효율
- 운영상태

작성 기준:
- 첫 줄에서 기간을 보여준다.
- 표 아래에는 전체 상태를 한 문장으로 정리한다.
- `운영상태`는 단순 라벨이 아니라, 왜 그 상태인지가 읽히게 써야 한다.
- `전월比`는 건수 행에는 증감값, 비율 행에는 퍼센트포인트 변화, 상태 행에는 유지/변경 사유를 적는다.
- 문구는 모두 검정색으로 쓰고, 강조를 위해 색을 바꾸지 않는다.
- 설명 문단이 여러 줄이어도, 같은 소제목 아래에서는 모든 설명 문단을 먼저 쓰고 그 다음 표를 둔다.

#### ☐ 접수채널별 처리현황

고정:
- `접수채널 / 전체 청구건수 / 자동지급건수 / 자동지급 비율 / 전월比`

가변:
- 채널별 수치
- 채널별 변화 해석 문구

반드시 들어갈 표 항목:
- 대면
- 비대면
- 실손24

작성 기준:
- 비율이 높은 채널과 전월 대비 변화가 큰 채널이 보이도록 정렬한다.
- 특정 채널이 튀면 그 이유를 한 줄로 설명한다.
- `전월比`에는 자동지급 비율의 전월 대비 변화(pp)를 적는다.

#### ☐ 진료구분별 처리현황

고정:
- `진료구분 / 자동지급건수 / 비율 / 전월比`

가변:
- 진료구분별 수치
- 진료구분 해석 문구

반드시 들어갈 표 항목:
- 입원
- 통원

작성 기준:
- 자동지급 건 중 어느 진료구분이 우세한지 읽히게 쓴다.
- 필요 시 운영 포인트를 한 줄로 덧붙인다.
- `전월比`에는 자동지급 비율의 전월 대비 변화(pp)를 적는다.

### 3-3. 2. 유형별 상세 현황

#### ☐ 자동지급 유형

고정:
- `순위 / 코드 / 유형명 / 비중 / 당월 / 전월 / 증감`

가변:
- 상위 5개 유형
- 각 유형의 비중과 증감
- 유형 변화 해석 문장

작성 기준:
- 당월 상위 유형을 우선 보여준다.
- 증감이 크거나 신규로 떠오른 유형은 해석에서 다시 언급한다.
- 유형명은 축약하지 않는다.
- 건수는 당월과 전월을 나란히 보여주고, 증감에는 건수 변화와 변화율을 함께 읽히게 쓴다.
- 표의 숫자도 모두 검정색이다.

#### ☐ 청구 특성 변화

고정:
- `청구 특성 / 주요 변화값 / 전월 자동지급률 / 당월 자동지급률 / 변화(pp) / 해석`

가변:
- `receipt_channel`, `treatment_type`, `hospital_type`, `diagnosis_code_primary`별 전월 대비 자동지급률 변화
- 변화가 큰 청구 특성 요소와 해석 문구

작성 기준:
- 청구 특성 요소가 전월 대비 자동지급률을 어떻게 바꿨는지 먼저 읽는다.
- 내부 segment 코드는 원문 기록으로만 남기고, 해석은 factor 요약을 우선한다.
- 변화가 작거나 근거가 약하면 `확인 필요`로 적는다.
- 한 문장에 하나의 해석만 담는다.

#### ☐ 자동지급 제외 유형

고정:
- `자동지급 불가한 조건에 해당하여 인심사 배분되는 유형 중 전월대비 삭감 / 부지급 비율이 5% 이상 낮아진 건`
- `순위 / 코드 / 유형명 / 비중 / 당월 / 전월 / 증감`

가변:
- 상위 5개 제외 유형
- 각 제외 유형의 비중과 증감
- 제외 유형 해석 문장

작성 기준:
- 제외 유형은 자동지급 유형과 분리해서 본다.
- 감소한 제외 유형이 있으면 운영 개선 신호로 볼 수 있도록 적는다.
- 원인 단정 대신 관찰된 사실과 운영 해석을 함께 쓴다.
- 건수 변화가 줄었는지, 유형 비중이 이동했는지를 함께 읽을 수 있게 쓴다.
- 유형명 컬럼만 왼쪽 정렬, 나머지는 가운데 정렬한다.

### 3-4. 3. Action Item

고정:
- `Action Item`
- 기본 체크 목록 골격

가변:
- 이번 달에 실제 확인해야 할 운영 액션
- 우선순위가 높은 후속 확인 항목

작성 기준:
- 액션은 2~3개 정도로 짧게 쓴다.
- 실행 가능한 문장으로 끝낸다.
- 운영관리자가 바로 다음 행동으로 옮길 수 있어야 한다.
- 액션 문구도 모두 검정색으로 유지한다.

## 4) 인사이트 작성 규칙

- 표 아래 설명은 숫자를 다시 반복하지 말고 해석을 붙인다.
- 변화가 없는 경우에도 `안정`을 명시한다.
- 청구 특성 변화는 `변화값 -> 전월 자동지급률 -> 당월 자동지급률` 순서로 해석하고, 사유는 짧게 붙인다.
- `전월比`는 단순 증감값이 아니라, 읽는 사람이 변화를 이해하도록 돕는 보조 정보로 쓴다.
- 한 문장에 하나의 해석만 담는다.

## 5) 제외 기준

- 청구유형 분류
- 최종 지급 여부 확정
- 지급금액 단독 확정
- 사람 심사를 자동지급으로 강하게 밀어주는 문구
- 근거 없는 원인 단정
- 수신자 개인정보 노출

## 6) 품질 체크포인트

- 제목과 발행년월만 봐도 기간이 바로 읽히는가
- 요약, 유형 상세, Action Item이 한 번에 분리되어 읽히는가
- 표의 숫자와 설명이 분리되어 있는가
- 파란색 가변영역이 데이터 중심으로 채워지는가
- 운영관리자가 상위 보고에 바로 옮길 수 있는가

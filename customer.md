# Customer — 자동심사 현황분석 Agent

## ICP
- **Primary**: 보험금 자동지급 운영 현황을 월말 기준으로 보고·해석해야 하는 운영관리자
- **Secondary**: 향후 자동지급 운영을 함께 보는 심사 Auditor, 심사담당자

## 핵심 불편사항
- 월말 기준 자동지급 대상군의 변화와 segment 변화를 사람이 직접 정리해야 한다.
- 전월 대비 비율 변화와 처리효율을 매번 수작업으로 비교해야 한다.
- 변화가 없더라도 `안정` 상태를 일관되게 보여주는 구조가 부족하다.
- 월간 리포트를 사람이 작성하고 발송해야 해서 보고 리드타임이 길다.
- 상위 보고용으로 설명 가능한 형태의 현황 요약을 매번 다시 가공해야 한다.

## 핵심 니즈
- 월말 스냅샷 기준으로 운영 현황을 한눈에 보고 싶다.
- 자동지급률, 성과, 안정 상태를 운영 효율 관점에서 읽고 싶다.
- 전월 대비 어떤 변화가 있었는지 빠르게 설명하고 싶다.
- segment가 어떻게 변했는지까지 근거 있게 보여주고 싶다.
- 월간 리포트를 자동으로 생성·발송하고, 다시 찾아볼 수 있어야 한다.

## 페르소나
| | A. 운영관리자 | B. 심사 Auditor | C. 심사담당자 |
|---|---|---|---|
| 핵심 고통 | 월말 변화/segment 변화/ROI 보고를 수작업으로 정리해야 함 | 운영 기준과 결과를 검토할 근거가 부족하면 점검이 어려움 | 향후 확대 시, 운영 현황을 빠르게 이해할 수 있어야 함 |
| 현재 대안 | 엑셀·쿼리·수작업 보고서 | 기존 보고서와 개별 확인 | 수동 확인, 구두 공유 |
| 필요한 것 | 운영모니터링, 리포트보관함, 안정 상태, 변화 설명 | 근거가 남는 현황 요약과 이력 | 이해하기 쉬운 현황 요약과 추후 확장 가능한 구조 |

## 고객이 기대하는 결과
- 운영 현황을 해석하는 시간이 줄어든다.
- 상위 보고용 정리 시간이 줄어든다.
- 보고서 생성과 발송의 수작업 부담이 줄어든다.
- 변화가 있을 때와 없을 때의 상태를 일관되게 확인할 수 있다.
